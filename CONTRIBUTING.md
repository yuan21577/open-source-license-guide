# 贡献指南

感谢你对本项目的关注！本项目是一个关于开源许可证合规与安全的学习型项目，欢迎任何形式的贡献。

---

## 你可以贡献什么

- **补充许可证信息**：如发现许可证对比表有误或有遗漏，欢迎提出
- **扩充 LICENSE_DB**：如果你知道某个常用 Python 包的许可证，欢迎在 `license_checker.py` 中补充
- **纠错与改进**：文档中的错别字、表述不清、链接失效等问题
- **提出新想法**：通过 Issue 提出你认为值得加入的内容方向

---

## 提交 Issue 的规范

请在 Issue 中包含以下信息：

1. **问题描述**：清晰说明你发现了什么问题，或你希望新增什么内容
2. **复现步骤**（如适用）：如果是 `license_checker.py` 的运行问题，请附上你的操作步骤
3. **预期行为 vs 实际行为**
4. **运行环境**：操作系统、Python 版本

---

## 提交 Pull Request 的规范

1. **Fork 本仓库**，在你的 Fork 上新建分支进行修改
2. **分支命名建议**：
   - 新功能：`feat/描述`，如 `feat/add-lgpl-support`
   - 文档修改：`docs/描述`，如 `docs/fix-comparison-table`
   - 问题修复：`fix/描述`
3. **Commit Message 格式**（参考 Conventional Commits 规范）：
   - `feat: 添加对 LGPL 许可证的风险提示`
   - `docs: 修正 comparison.md 中的兼容性描述`
   - `fix: 修复包名大小写匹配问题`
4. 提交 PR 时，请在描述中说明**做了什么、为什么这样做**

---

## 行为准则

本项目遵循开放、友善、尊重的社区氛围。请在交流中保持专业与善意，共同维护一个健康的学习环境。
